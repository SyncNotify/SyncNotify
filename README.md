
SyncNotify 是一款专为中小学课堂教学设计的轻量级、无服务器（Serverless）信息同步与通知工具。

## 💡 项目背景与痛点

在中小学教学过程中，教室内往往只配备了连接校园网或外网的 Windows 电脑（多媒体白板）。当老师不在教室（如在办公室、操场，或由其他老师代课）时，临时通知、倒计时提醒等信息往往传达不及时。

同时，在中学班级环境中，**部署和维护一套独立的云端服务器非常不现实**（涉及公网IP、服务器成本、校园网防火墙、网络维护等问题）。

**SyncNotify 巧妙地解决了这个问题：** 它完全**不依赖服务器**，而是利用现有的**同步网盘**（如坚果云、OneDrive、天翼云盘等）作为数据中转桥梁。

**📌 本项目诞生于我的高中时代，目前已圆满完成历史使命。作者已毕业，项目停止功能更新，仅作展示。如果未来有机会可能会重构本项目。**

## 🛠️ 工作原理

项目的核心逻辑是：**“网盘同步文件 + 本地目录监听 = 实时通知”**。

<img width="2111" height="110" alt="af174508836dae71921ab03a1aeaf652" src="https://github.com/user-attachments/assets/cfbdfc5a-2b05-47f1-ac93-0af0279ba325" />


1. **发送端（老师）：** 老师在手机上使用特定软件，将包含通知内容的 `JSON` 或 `TXT` 文件保存到同步网盘的指定目录。
2. **云端同步：** 同步网盘的电脑客户端自动将该文件下载到教室的电脑上。
3. **接收端（本软件）：** `SyncNotify` 运行在 Windows 后台，**实时监听**目标同步目录。一旦检测到文件新建或更改，立即解析文件内容，并根据指令触发相应的通知或计时器。

---

## ✨ 核心功能

* **⚡ 零服务器依赖：** 仅需一个同步网盘账号，即可实现跨设备、跨网络的“伪即时通讯”，极大地降低了部署门槛。
* **📂 双格式兼容：** 支持直接解析纯文本（`TXT`）和结构化数据（`JSON`）。
* **📱 强大的 Android 联动扩展：** 当配合 Android 端发送精心定制的 `JSON` 配置文件时，Windows 端可解锁以下高级功能：
    * **定时弹出：** 支持设定未来的特定时间点自动触发通知。
    * **多样化显示：** 支持**悬浮窗**（不干扰当前授课）或**全屏弹出**（强力提醒）两种展示方式。
    * **声音辅助：** 可配置弹窗时是否播放提示音。
    * **实名通知：** 动态显示发送者（老师）的姓名，让学生明确消息来源。

## 🚀 技术栈

* **框架：** .NET / C# WPF (Windows Presentation Foundation)
* **架构模式：** MVVM (Model-View-ViewModel)
* **核心技术：** `FileSystemWatcher`（文件系统监听）、JSON 序列化/反序列化

## ⚙️ 快速上手

### 1. 准备工作
* 在教室 Windows 电脑和老师的手机上安装同一种同步网盘（如坚果云、OneDrive 等），并登录同一账号。
* 在网盘中创建一个专属文件夹（例如：`D:\SyncNotify\Message\`），确保手机端和电脑端均能同步该目录。

### 2. 配置与运行
1. 下载并运行 `SyncNotify.exe`。
2. 在软件设置中，将**监听目录**设置为上述网盘的同步文件夹路径。
3. 点击“启动监听”（或设置开机自动运行并监听）。

### 3. 发送消息示例 (JSON)
当通过 Android 端发送类似以下结构的 `JSON` 文件至同步目录时，软件将完美解析：

```json
{
  "sender": "张老师",
  "content": "请各科课代表马上到办公室抱作业！",
  "type": "fullscreen", 
  "hasSound": true,
  "playAt": "2026-05-31 14:00:00"
}
